

# پیکربندی VS Code 

برای تبدیل شدن به یک برنامه‌نویس حرفه‌ای، محیط توسعه (IDE) شما باید هوشمند و کارآمد باشد. در این بخش ابزارهای کلیدی (VS Code) را تنظیم می‌کنیم.

## ۱. افزونه‌های ضروری (Extensions)
وارد بخش Extensions (`Ctrl + Shift + X`) شوید و افزونه‌های زیر را نصب کنید:
- **پکیج Python:** برای برجسته‌سازی سینتکس، عیب‌یابی (Debugging) و تکمیل خودکار کد (IntelliSense).
- **پکیج Pylance:** موتور تحلیل کد فوق‌العاده سریع مایکروسافت که خطاهای نگارشی و تایپی پایتون را قبل از اجرا به شما نشان می‌دهد.
- **پکیج GitLens:** قدرتمندترین ابزار برای مشاهده تاریخچه گیت. به شما نشان می‌دهد هر خط کد را چه کسی، در چه تاریخی و با چه کامیتی تغییر داده است.

## ۲. تنظیمات (Settings.json)
برای بالا رفتن کیفیت کد، تنظیمات زیر را اعمال کنید. کلیدهای `Ctrl + ,` را بزنید، روی آیکون لایه (Open Settings JSON) در بالا سمت راست کلیک کنید و تنظیمات زیر را اضافه کنید:
```json
{
    "editor.formatOnSave": true,
    "editor.renderWhitespace": "all",
    "python.analysis.typeCheckingMode": "basic",
    "files.trimTrailingWhitespace": true,
    "files.insertFinalNewline": true
}
```
- ** گزینه Format on Save**: با هر بار ذخیره (Ctrl + S)، کدهای شما طبق استاندارد مرتب می‌شوند.
- ** گزینه Insert Final Newline**: به طور خودکار یک خط خالی استاندارد به انتهای فایل‌ها اضافه می‌کند (یک اصل مهم در گیت).

## بررسی، عیب‌یابی و تکمیل کد دفترچه تلفن
کد فعلی ما چند مشکل اساسی (Bug) و دو بخش ناتمام (TODO) دارد. ابتدا نگاهی به وضعیت فعلی کد می‌اندازیم و سپس آن را اصلاح می‌کنیم.
1. **باگ فایل خالی**: اگر فایل ```data/phonebook.txt``` وجود نداشته باشد یا خالی باشد، متد ```read_file_contacts()``` با خطای ```ValueError``` مواجه می‌شود چون خط خالی را نمی‌توان بر اساس -- به دو قسمت تقسیم کرد.

2. **باگ کرش در منو**: اگر کاربر ورودی اشتباهی بدهد یا در بخش جستجو نام کاربری وجود نداشته باشد، کل برنامه داخل بلاک ```except``` می‌افتد و دلیل خطا مشخص نمی‌شود.

3. **پیاده‌سازی موارد ناتمام (TODO)**: عدم بررسی تکراری بودن مخاطب و عدم جلوگیری از ورود کاراکتر جداکننده (--).




# معرفی JSON

---

## JSON چیست؟

**JSON** (JavaScript Object Notation) یک فرمت استاندارد و سبک برای ذخیره و انتقال داده‌ها است.

ویژگی‌ها:
- خوانایی بالا برای انسان
- پردازش آسان توسط کامپیوتر
- مستقل از زبان برنامه‌نویسی
- پرکاربرد در APIها و سرویس‌های وب

---

## چرا از JSON استفاده می‌کنیم؟

قبل از JSON، انتقال داده‌ها معمولاً با XML انجام می‌شد که ساختار پیچیده‌تری داشت.

مزایای JSON:
- حجم کمتر
- خوانایی بیشتر
- سرعت پردازش بالاتر
- پشتیبانی توسط تقریباً تمام زبان‌های برنامه‌نویسی

---

## ساختار کلی JSON

JSON از دو ساختار اصلی تشکیل شده است:

- Object (شیء)
- Array (آرایه)

نمونه:

```json
{
  "name": "Ali",
  "age": 22,
  "student": true
}
```

---

# Object

Object مجموعه‌ای از کلید و مقدار (Key-Value) است.

قواعد:

- داخل `{ }`
- کلیدها همیشه رشته هستند.
- هر کلید با `:` از مقدار جدا می‌شود.
- اعضا با `,` جدا می‌شوند.

مثال:

```json
{
  "name": "Sara",
  "age": 25,
  "city": "Tehran"
}
```

---

# انواع داده در JSON

JSON از انواع داده زیر پشتیبانی می‌کند:

| نوع | مثال |
|------|------|
| String | `"Hello"` |
| Number | `25` |
| Boolean | `true` |
| Null | `null` |
| Object | `{}` |
| Array | `[]` |

مثال:

```json
{
    "name": "Reza",
    "age": 20,
    "isStudent": true,
    "score": null
}
```

# JSON در برنامه‌نویسی

تقریباً همه زبان‌ها از JSON پشتیبانی می‌کنند.

مثال در Python:

```python
import json

text = '''
{
    "name": "Ali",
    "age": 22
}
'''

data = json.loads(text)

print(data["name"])
```

---

مثال در JavaScript

```javascript
const person = {
    name: "Ali",
    age: 22
};

const jsonText = JSON.stringify(person);

console.log(jsonText);
```
